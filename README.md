# 角色对话统计工具

一个用于分析跑团日志文件的工具，自动统计每个角色的发言数、字数、场外发言等，并生成详细的 Excel 和 CSV 报告。

## ✨ 特性

- 🔍 **智能格式检测**：自动识别4种常见日志格式
- 📊 **详细统计**：发言数、字数、平均字数、场外发言统计
- 📈 **多种输出**：生成 Excel 和 CSV 双格式报告
- 📅 **智能分天**：支持分隔符检测和长时间间隔分天，解决熬夜开团问题
- 🎨 **美观报表**：Excel 包含样式设计，场外发言黄色标识
- 🚀 **一键运行**：自动处理依赖和环境配置

## 📦 快速开始

### Windows 用户（推荐）

1. 将日志文件（`.txt` 格式）放入项目根目录或 `logs/` 文件夹
2. 双击运行 `run.ps1` 或在 PowerShell 中执行：
   ```powershell
   .\run.ps1
   ```
3. 按提示选择日志文件（输入文件编号）
4. 查看生成的 CSV 和 Excel 报告

### 直接运行

```bash
python\python.exe app/main.py
```

## 📋 支持的日志格式

工具会自动检测以下4种格式：

1. **Format1**: `用户名(ID) YYYY-MM-DD HH:MM:SS`
   ```
   张三(12345) 2025-05-24 20:30:15
   这是一条消息
   ```

2. **Format2**: `用户名(ID) YYYY/MM/DD HH:MM:SS`
   ```
   张三(12345) 2025/05/24 20:30:15
   这是一条消息
   ```

3. **Format3**: `HH:MM:SS <用户名>消息内容`
   ```
   20:30:15 <张三>这是一条消息
   ```

4. **Format4**: `HH:MM:SS<用户名>:消息内容`
   ```
   20:30:15<张三>:这是一条消息
   ```

## 🎯 核心功能

### 智能分天逻辑

采用混合策略，解决熬夜开团跨天问题：

1. **优先级1：分隔符检测**
   - 检测 `.log off`、`===日志开始===` 等明确分隔符
   - 支持中文句号：`。log off`

2. **优先级2：长时间间隔检测**
   - 默认：超过 **6小时** 无发言视为新的一天
   - 可配置阈值（修改 `app/parser/log_parser.py` 第9行）

### 自动文件命名

输出文件自动使用输入文件名：

```
输入: logs/传说之路第1场 2025.05.24.txt
输出:
  ├─ 传说之路第1场 2025.05.24.csv
  └─ 传说之路第1场 2025.05.24.xlsx
```

### 场外发言识别

自动识别带括号的场外发言（OOC）：
- 中文括号：`（这是场外发言）`
- 英文括号：`(这是场外发言)`

在 Excel 报告中使用黄色背景标识。

## 📊 输出报告

### CSV 文件
包含总体统计和按日统计的表格数据。

### Excel 文件
- **统计工作表**：总体统计 + 每日统计
- **角色对话工作表**：每个角色的完整对话内容
- **样式设计**：表头加粗、边框、场外发言黄色背景

## ⚙️ 配置

### 调整分天时间阈值

编辑 `app/parser/log_parser.py` 第9行：

```python
TIME_GAP_THRESHOLD_HOURS = 6  # 超过6小时无发言视为分天
```

建议值：
- `3`：适合短时段开团
- `6`：默认值，适合大多数情况
- `12`：适合长时段开团
